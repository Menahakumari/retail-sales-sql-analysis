## Dataset Overview

This project analyzes a retail sales dataset using SQLite. The dataset contains transaction details such as customer information, product category, quantity, price, and total sales amount.

---

## Table Used

**retail_sales**

* transaction_id
* transaction_date
* customer_id
* gender
* age
* product_category
* quantity
* price_per_unit
* total_amount

---

## Query Explanations

### 1. View Sample Records

```sql
SELECT * FROM retail_sales LIMIT 10;
```

Displays a small sample of records to understand the structure and values in the dataset.

---

### 2. Count Total Records

```sql
SELECT COUNT(*) AS total_records FROM retail_sales;
```

Counts the total number of rows to verify that all CSV records were imported correctly.

---

### 3. Filter Data by Product Category

```sql
SELECT * FROM retail_sales WHERE product_category = 'Electronics';
```

Retrieves only those transactions that belong to the Electronics category.

---

### 4. Top Sales Transactions

```sql
SELECT transaction_id, product_category, total_amount
FROM retail_sales
ORDER BY total_amount DESC
LIMIT 5;
```

Finds the top 5 highest-value transactions based on total sales amount.

---

### 5. Total Sales by Category

```sql
SELECT product_category, SUM(total_amount) AS total_sales
FROM retail_sales
GROUP BY product_category;
```

Calculates total revenue generated by each product category.

---

### 6. Average Sales per Category

```sql
SELECT product_category, AVG(total_amount) AS avg_sales
FROM retail_sales
GROUP BY product_category;
```

Computes the average transaction value for each product category.

---

### 7. Transaction Count by Gender

```sql
SELECT gender, COUNT(*) AS transaction_count
FROM retail_sales
GROUP BY gender;
```

Shows how many transactions were made by each gender group.

---

### 8. Categories with High Sales (HAVING)

```sql
SELECT product_category, SUM(total_amount) AS total_sales
FROM retail_sales
GROUP BY product_category
HAVING total_sales > 100000;
```

Filters grouped results to show only categories with sales exceeding 100,000.

---

### 9. Monthly Sales Report (BETWEEN)

```sql
SELECT SUM(total_amount) AS monthly_sales
FROM retail_sales
WHERE transaction_date BETWEEN '2023-01-01' AND '2023-01-31';
```

Calculates total sales within a specific date range for monthly analysis.

---

### 10. Customer Pattern Search (LIKE)

```sql
SELECT * FROM retail_sales
WHERE customer_id LIKE 'CUST%';
```

Finds all customer records whose IDs start with the prefix "CUST".

---

## Conclusion

These queries help explore, summarize, and analyze retail sales data using core SQL concepts such as filtering, sorting, aggregation, grouping, and pattern matching. The results can be exported as CSV files for reporting and documentation.
